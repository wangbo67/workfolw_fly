# 代码上下文分析

在生成测试之前，必须读取目标代码引用的所有类型。使用错误的构造器、遗漏必填字段或创建无效对象的测试，将在编译时报错或产生无意义的结果。

## 写测试前需要读取什么

读取目标类后，识别并读取以下内容：

1. **直接参数类型**：方法参数中使用的每个类
2. **返回类型**：被测方法返回的每个类
3. **构造器注入的字段类型**：需要 mock 的依赖
4. **领域实体 / DTO**：方法体中创建或转换的类
5. **枚举**：在条件判断、switch 语句或参数中使用的枚举
6. **自定义异常**：方法中抛出的异常类
7. **校验器 / 约束**：测试校验逻辑时涉及的自定义注解类

## 模块模式下的上下文分析

当目标为模块/包时，上下文分析需要额外考虑：

1. **共享依赖缓存**：同一模块内多个类可能引用相同的依赖（DTO、枚举等），只读取一次并复用，避免重复读取
2. **内部依赖识别**：模块内的类之间可能互为依赖（如 Service 依赖同模块的 Repository），在拓扑排序后先分析底层类，其依赖信息可直接供上层类使用
3. **跨模块依赖**：如果依赖不在当前模块内，仍需读取，但不需要为其生成测试
4. **依赖收集策略**：
   - 第一遍：扫描所有目标类的 import 语句，建立完整的依赖清单
   - 第二遍：按拓扑顺序逐类读取依赖，已读取的跳过
   - 去重：相同类只读取一次，结果在类间共享

## 为什么这很重要

```java
// 没有读取依赖 — 构造器参数错误
var request = new OrderRequest("product-1", 5);
// 实际构造器：OrderRequest(String productId, int quantity, String customerId)

// 读取了依赖 — 正确
var request = new OrderRequest("product-1", 5, "customer-123");
```

## 如何高效读取依赖

1. 读取目标类的 import 语句，识别引用的类型
2. 使用 Glob 查找源文件：`**/OrderRequest.java`
3. 读取每个依赖，了解：
   - 构造器参数（类型和顺序）
   - 必填字段与可选字段
   - Builder 模式（如果有 — 使用 Builder）
   - 工厂方法（如果有 — 优先于构造器）
   - 可用的枚举值

## 特别注意

- **Lombok 注解**：`@Builder`、`@AllArgsConstructor`、`@RequiredArgsConstructor` 会改变对象的创建方式
- **校验注解**：字段上的 `@NotNull`、`@NotBlank`、`@Size` 指明了约束条件
- **继承关系**：如果类继承了父类，也需要读取父类
- **泛型**：理解类型参数，确保在测试中使用正确的类型

## 检查清单

在编写任何测试方法之前：
- [ ] 读取目标方法使用的所有参数类型
- [ ] 读取所有返回类型
- [ ] 读取方法体中创建或修改的领域实体
- [ ] 读取条件判断中使用的枚举类
- [ ] 识别构造器、Builder 或工厂方法

**模块模式额外检查：**
- [ ] 已缓存所有已读取的依赖，避免重复读取
- [ ] 已识别模块内类间的内部依赖关系
- [ ] 按拓扑顺序分析，底层类的依赖信息已供上层类复用
